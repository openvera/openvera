# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in OpenVera, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, send an email to **security@openvera.dev** with:

- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will acknowledge your report within 48 hours and work with you to understand and address the issue.

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest  | Yes       |

## Scope

This policy applies to the OpenVera codebase and its default configuration. Third-party dependencies are managed through regular updates.
